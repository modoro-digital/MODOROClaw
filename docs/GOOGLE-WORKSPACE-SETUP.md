# Google Workspace setup: where `client_secret.json` comes from

9BizClaw uses `gogcli` for Google Workspace access. The JSON file is not generated by this repo and it is not a Google Workspace Admin export. It is an OAuth Client ID file that must be created in Google Cloud Console for each customer/project.

## What file to upload

Upload an OAuth Client ID JSON for a Desktop app.

The file usually looks like:

```json
{
  "installed": {
    "client_id": "...apps.googleusercontent.com",
    "client_secret": "...",
    "auth_uri": "https://accounts.google.com/o/oauth2/auth",
    "token_uri": "https://oauth2.googleapis.com/token"
  }
}
```

Do not upload:

- Service Account JSON files
- API key JSON files
- Google Workspace Admin export files
- The token file created after OAuth login

## One-time Google Cloud setup

1. Open [Google Cloud Console Credentials](https://console.cloud.google.com/apis/credentials).
2. Create or select the customer project.
3. Enable the APIs that 9BizClaw will use:
   - Google Calendar API
   - Gmail API
   - Google Drive API
   - People API
   - Google Tasks API
   - Google Sheets API
   - Google Docs API
   - Apps Script API
4. Open [Google Auth Platform](https://console.cloud.google.com/auth/overview) and configure the OAuth consent screen.
5. If the app is still in Testing, add the CEO's Google account under Test users.
6. Open [Google Auth Clients](https://console.cloud.google.com/auth/clients).
7. Click Create Client.
8. Set Application type to Desktop app.
9. Name it, for example `9BizClaw Desktop`.
10. Download the JSON file. Google usually names it like `client_secret_...apps.googleusercontent.com.json`.

## 9BizClaw setup

1. Open Dashboard > Google Workspace > Settings.
2. Upload the downloaded OAuth Client JSON.
3. Enter the Google email that should be connected.
4. Click Connect.
5. Approve the Google OAuth screen in the browser.

Raw `gogcli` equivalent for development:

```bash
gog auth credentials ./client_secret_....json
gog auth add ceo@company.com --services calendar,gmail,drive,contacts,tasks,sheets,docs,appscript
```

For AppSheet-style workflows, this integration can read and update the Google Sheets that AppSheet uses as data sources. Direct AppSheet app/admin API operations are not wrapped yet.

## Common setup problems

| Problem | Cause | Fix |
|---|---|---|
| "Wrong JSON type" | Uploaded a Service Account JSON or another Google JSON file | Create OAuth Client ID with Application type: Desktop app |
| Access denied during login | OAuth app is in Testing and the Google account is not a test user | Add the account under Test users |
| API has not been used in project | Required API was not enabled in the same project that owns the OAuth client | Enable the missing API and retry after propagation |
| Needs reconnect after 7 days | OAuth app is External and still in Testing | Move the OAuth app to Production for long-lived refresh tokens |
